On October 10th, Greg Morefield, Don Oliver, Tom Lutz, Jason Steinbrunner, Mark Apgar, Brian Bahr, BJ McClure, and Brian Samson traveled to Microsoft Developer Days 2000 at the Radisson Hotel in Sharonville.  The goal of this conference was to provide developers an architectural and implementation-based understanding of the Microsoft .NET initiative, and how to use the new Microsoft 2000 lineup to build “world class applications”.

At the beginning of the day, we were unsure as to how much techno-geek material vs. Microsoft marketing we were going to get.  We were hoping for code snippets, descriptions of resolutions to past development issues, and an idea of what technical value the new Microsoft 2000 lineup would provide.  By 5:00pm, we felt that although there definitely was a significant amount of MS hype sprinkled throughout the day, some of the material indeed educated us on how we might be using the .NET lineup in the near future. 

First, let’s define what the .NET initiative actually is.  The goal of .NET is to provide the capability “to create programs that transcend device boundaries utilizing the Internet”.  To this point, application development has been geared towards integrating specific applications together – in other words, a tightly coupled development.  Microsoft proposes that new development center around a loosely coupled development methodology, where application components act as web services.  These services expose their features programmatically using standard Internet protocols such as HTTP and XML. Conceptually, this methodology is nothing more than black-box component development with exposed API interfaces, so it shouldn’t be too difficult to retrain yourself. 

Now, of course Microsoft isn’t providing all of this nifty information and methodology without new tools to help you out, so here’s the run-down of new .NET Enterprise Servers you’ll be seeing soon:

·         Microsoft Windows Server 2000 – from the “duh” category, this is Microsoft’s bread and butter operating system that’s required for anything else to work.

·         Microsoft Commerce Server 2000 – the old Site Server Commerce Edition.  This solution allows you to quickly build an online business.

·         Microsoft BizTalk Server 2000 – the center of the parade.  This tool lies at the heart of providing a loosely coupled environment, providing an integrated solution for workflow orchestration.

·         Microsoft Exchange Server 2000 – adds stronger workflow collaboration and now provides access capabilities outside of the standard PC.

·         Microsoft Application Server 2000 – a new tool for deploying and managing high-availability web applications.  Provides component load balancing, advanced clustering, and rolling upgrades.

·         Microsoft Host Integration Server 2000 – allows you to connect to existing other systems (mainframes, unix boxes, etc.).  Provides MMC support, single sign-on, and automatic password synchronization capabilities.

·         Microsoft Internet Security and Acceleration Server 2000 – otherwise known as Proxy Server.

·         Microsoft SQL Server 2000 – I don’t believe SQL Server 2000 is a required component, but it’s in production today, so I thought I’d throw it in for realism.

 

The following is a brief summary of the tracks the group attended.  If you have any questions about these Microsoft products or the new development focus, ask someone from this group and they’ll be more than happy to expound upon their knowledge.  We also have the conference CD available in the office if you care to review the presentations.

It was demonstrated how BizTalk can now handle workflow and automation situations, and changing that workflow can theoretically be as simple as modifying a flowchart (which looks strangely similar to Visio).  Microsoft affectionately terms this capability orchestration, where each step of a process is initiated by the receipt of an XML message from a particular location, and each step ends by sending an XML message to a particular location.

 

Implementation Track (Mark Apgar)

This session was divided into 4 main categories: Commerce Server 2000 Solution Sites, SQL Server 2000 XML support, Exchange Server 2000, and using the SOAP Toolkit for Visual Studio 6.0 to create Web Services.  Commerce Server 2000 Solution sites provide a starting point for the developers.  The solution sites include a retail site, an active directory supplier site, and a blank site.  These solution sites provide basic framework for developing internet sites including a global.asa file, ASP files, and templates in a virtual root, defined pipelines, set up of SQL server and related tables.  SQL Server 2000 XML support provides a way for the developer to query the database and return the information in XML format.  Extensions of ADO were added to allow the user to query the database and return a query stream in XML format.  Exchange Server 2000 Web Store is a good place to hold Corporate Knowledge where the application can access it.  The request would be made using XML to the Web Store where HTTP-DAV would be used to search and return the information in XML format to the browser.  Finally, the SOAP Toolkit for Visual Studio 6.0 is a general XML messaging protocol that is OS, project model, and programming language neutral.  The basic use would be for retrieving information from another web site without leaving the current site.  For example, if a user would like to track the order status of an article of clothing bought on the web, the website would have the ability to send a request to UPS, return data to the website and display the status of the order without leaving the current page.  This process describes a web service.

The second half of the track focused on how XML and BizTalk Server help integrate applications and business processes.  The presentation detailed the process of using the server including document formats, data mapping, submitting documents, processing and routing, and business process Orchestration.  The presentation went into depth on each of these processes and integrating them with SQL Server and XML.
